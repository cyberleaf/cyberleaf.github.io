Wordpress vs. Jekyll etc.

Zur Verwaltung der Inhalte einer Website ist Wordpress wohl eines der am häufigsten eingesetzten Software-Tools, insbesondere bei der Erstellung und Pflege eines Blogs. Der große Vorteil liegt zweifellos darin, dass man Inhalte einstellen kann, ohne sich mit dem Layout der Seite beschäftigen zu müssen. Durch die Wahl eines Themes ist das Aussehen im wesentlichen vorgegeben und der Autor kann sich ganz dem Inhalt widmen. Die Hürde im Netz zu publizieren, ist dadurch für weniger technikaffine Menschen stark herabgesetzt, diese sind dank solcher Tools wie Wordpress in der Lage ihre Inhalte einer breiten Öffentlichkeit in ansprechender Form zugänglich zu machen, zweifellos in vielen Fällen eine Bereicherung der Netzkultur. Dennoch hat Wordpress auch einige Schwäche. So reicht ein einfacher Webserver für das Betreiben eines Wordpress-Blogs nicht aus, es muss die Kombination aus Apache-Server, MySQL-Datenbanksystem und PHP (zusammen mit einem Linux-Betriebssystem auch als LAMP bezeichnet) vom Hoster angeboten werden. PHP und MySQL sorgen jedoch für eine hohe Serverlast bei starkem Traffic, muss doch jede Seite für jeden Aufruf erneut von PHP generiert werden. Weiterhin ist die Installation von Wordpress zwar einfach, aber das Update auf eine neue WP-Version bei bereits bestehendem Blog kann seine Tücken haben. Deshalb scheuen viele Anwender diesen Schritt, was wiederum ein Sicherheitsrisiko darstellt.
